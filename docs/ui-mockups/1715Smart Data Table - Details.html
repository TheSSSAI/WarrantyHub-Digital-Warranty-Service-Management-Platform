Component Name

Smart Data Table

Component Type

organism

Description

A comprehensive data display component supporting sorting, filtering, selection, and responsive adaptation. It handles complex data sets with states for loading, error, and empty results, transforming into a card-based layout on mobile devices for optimal usability.

Design System Alignment

Utilizes the grid system for layout, typography scale for headers and body text, semantic colors for status badges, and standard spacing tokens. Integrates molecular components like Pagination and Filter Bar.

Responsive Specifications

Mobile Adaptations

  • Table headers hidden
  • Rows transformed into independent cards
  • Data labels (data-label attribute) displayed inline
  • Actions moved to accessible touch targets
  • Horizontal scrolling disabled in favor of vertical stack

Tablet Adaptations

  • Horizontal scroll enabled for wide tables
  • Sticky first column for context preservation
  • Condensed padding

Desktop Enhancements

  • Full grid layout
  • Sticky header row
  • Hover effects on rows
  • Bulk action toolbar visibility

Accessibility Features

Keyboard Support

Sort headers focusable via Tab, Enter to sort. Action buttons keyboard accessible. Pagination navigable.

Screen Reader Support

Semantic

structure on desktop. Role='row/gridcell' attributes maintained. Sort direction announced via aria-sort.

Focus Management

Visible focus rings on all interactive elements within cells.

Aria Attributes

  • aria-sort
  • aria-label
  • aria-selected
  • aria-busy (loading state)

Color Contrast

Text and icons meet 4.5:1 ratio against row backgrounds.

Implementation Notes

Embedded Css Architecture

BEM naming convention used (.smart-table__element--modifier). CSS Variables used for theming.

Component Organization

Separated into Container, Toolbar (Filter/Search), Table Content, and Pagination Footer.

Responsive Design

CSS Grid/Flexbox used for layout. Media queries handle the table-to-card transformation.

Accessibility Compliance

Includes scope attributes on th, proper ARIA labels on pagination.

Browser Support

Modern browsers (Chrome, Firefox, Safari, Edge). Flexbox and Grid support required.

Usage Guidelines

When To Use

Displaying collections of structured data (users, logs, products) that require management (sort/filter/edit).

Content Guidelines

Keep column headers concise. Truncate long text in cells. Use badges for status.

Placement Rules

Main content area of dashboards or management pages.

Customization Options

  • .smart-table--dense (Compact view)
  • .smart-table--responsive (Enable mobile card view)
  • .smart-table--sticky-header

Testing Recommendations

Browser Testing

Verify sticky positioning in Safari and Firefox. Check overflow behavior on Edge.

Device Testing

Test card view transformation on iPhone SE and Pixel devices.

Accessibility Testing

  • Navigate table using Tab/Shift+Tab
  • Verify screen reader announces column headers correctly
  • Check contrast of zebra striping if used

Performance Testing

Test rendering speed with 50+ rows. Check scroll performance.
