Component Id

Component Name

Status Timeline

Component Type

organism

Description

A responsive tracking component visualizing the lifecycle of a service request, displaying chronological states with timestamps, descriptions, and contextual actions.

Design System Alignment

Utilizes primary colors for active states, success colors for completion, and neutral tokens for pending states. Implements typography scale for hierarchy and spacing tokens for vertical rhythm.

Design Tokens Used

--color-success-main
--color-gray-200
--color-gray-900
--spacing-4
--font-size-md
--radius-md

Responsive Specifications

Mobile Adaptations

  • Timeline strictly vertical for readability on narrow screens
  • Connectors scale vertically to fit content height
  • Actions wrap below description

Tablet Adaptations

  • Maintains vertical layout for information density
  • Timestamps may align to right side

Desktop Enhancements

  • Optional horizontal layout for wizard-like steps
  • Expanded width for description content

Accessibility Features

Keyboard Support

Action buttons within timeline are tab-navigable

Screen Reader Support

Uses ordered list

    for semantics, aria-current='step' for active item

Focus Management

Visible focus rings on interactive elements

Aria Attributes

  • aria-label='Service Request Progress'
  • aria-current='step'

Color Contrast

Text and icons meet 4.5:1 ratio, status conveyed via text and icons, not just color

Implementation Notes

Embedded Css Architecture

BEM naming convention scoped to .timeline component

Component Organization

Structured list items with modular content slots

Responsive Design

Flexbox layout handling both vertical and conditional horizontal flow

Accessibility Compliance

Semantic list structure with visual indicators supported by text

Usage Guidelines

When To Use

To display the history and current status of a process like a service request or delivery.

Content Guidelines

Titles should be short status names. Descriptions provide context (who/why). Timestamps are critical for history.

Placement Rules

Main content area of detail screens.

Customization Options

  • Add .timeline--horizontal-desktop for stepper-like visualization
  • Inject action buttons into .timeline__actions

Testing Recommendations

Browser Testing

Verify connector line alignment across Chrome, Safari, Firefox

Device Testing

Test text wrapping on small mobile screens (iPhone SE)

Accessibility Testing

  • Verify screen reader announces step number and status
  • Check color contrast of gray timestamps
  • Ensure focus order follows timeline logic

Performance Testing

Check rendering with many (10+) timeline items
