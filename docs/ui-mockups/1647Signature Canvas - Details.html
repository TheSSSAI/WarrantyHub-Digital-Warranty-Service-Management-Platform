Component Information

Component Name

Component Type

organism

Description

Interactive input surface for capturing digital signatures with support for clear/save actions and orientation adjustments.

Design System Alignment

Uses form_inputs.signature_canvas tokens, primary colors for active states, and standard spacing units.

Design Tokens Used

--color-primary-600
--color-gray-200
--color-gray-50
--radius-lg
--spacing-md
form_inputs.signature_canvas

Responsive Specifications

Mobile Adaptations

  • Canvas height adjusts based on orientation
  • Touch-action: none prevents page scrolling while signing
  • Buttons sized for 44px touch targets

Tablet Adaptations

  • Increased canvas width to fill container
  • Maintains aspect ratio for high-fidelity capture

Desktop Enhancements

  • Mouse cursor changes to crosshair
  • Focus states enhanced for keyboard users

Accessibility Features

Keyboard Support

Clear/Save buttons accessible via Tab

Screen Reader Support

Canvas has aria-label describing content; status messages announced

Focus Management

Visual focus ring on canvas container when active

Aria Attributes

  • aria-label='Signature Pad'
  • aria-disabled
  • role='application'

Color Contrast

Placeholder text meets 4.5:1, Borders meet 3:1 against background

Implementation Notes

Embedded Css Architecture

BEM methodology used for component scoping. CSS variables for theming.

Component Organization

Self-contained .signature-pad block with internal elements for label, area, and actions.

Responsive Design

Uses media queries for orientation changes to adjust canvas height.

Accessibility Compliance

Includes fallback content for canvas and proper ARIA labels.

Browser Support

Works in all modern browsers supporting HTML5 Canvas.

Usage Guidelines

When To Use

When capturing user authorization or verification (e.g., job completion, delivery receipt).

Content Guidelines

Label should clearly state what is being signed for.

Placement Rules

Place at the bottom of forms, just before submission actions.

Customization Options

  • .signature-pad--signed : changes visual state to solid border
  • .signature-pad--locked : disables interaction
  • .signature-pad--error : highlights border in red

Testing Recommendations

Browser Testing

Chrome, Safari (iOS), Firefox

Device Testing

Test touch events on iPad and Android phones in both orientations

Accessibility Testing

  • Verify tab order includes action buttons
  • Check contrast of placeholder text

Performance Testing

Ensure canvas drawing is smooth (60fps) on lower-end mobile devices
