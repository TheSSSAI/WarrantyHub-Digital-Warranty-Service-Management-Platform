Component Name

Component Type

organism

Description

A comprehensive data visualization container capable of rendering various chart types (Bar, Line, Pie, Funnel) with built-in filtering controls, legends, and responsive states. It encapsulates the 'Card' molecule and handles loading, empty, and populated states.

Design System Alignment

Utilizes the Card molecule for containment, Typography tokens for hierarchy, and Semantic colors for data visualization.

Responsive Specifications

Mobile Adaptations

  • Filters stack vertically above the chart
  • Chart height reduces to preserve viewport space
  • Tooltips triggered on tap instead of hover
  • Legend moves below chart

Tablet Adaptations

  • Filters align inline with title where space permits
  • Standard chart aspect ratio maintained

Desktop Enhancements

  • Hover interactions for tooltips
  • Side-by-side legend positioning for complex charts
  • Expanded filter options visible

Accessibility Features

Keyboard Support

Focusable chart elements, skip links for data tables

Screen Reader Support

Charts include role='img' with aria-label describing trends. Data is backed by visually hidden semantic tables.

Focus Management

Visible focus rings on filter controls

Aria Attributes

  • aria-label
  • role='graphics-document'
  • aria-hidden='true' (for decorative elements)

Color Contrast

Data colors verified against background; Text labels meet 4.5:1 ratio

Usage Guidelines

When To Use

Displaying aggregated data trends, comparisons, or distributions on Dashboards.

Content Guidelines

Title should be descriptive. Legends used when >1 data series. Tooltips for precise values.

Placement Rules

Main content area of Dashboard layouts. Can span 12 (full), 6 (half), or 4 (third) columns.

Customization Options

  • Chart Type (Bar, Line, Pie, Funnel)
  • Height (fixed or fluid)
  • Legend position (Top, Bottom, Right)
