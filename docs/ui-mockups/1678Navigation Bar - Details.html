Component Information

Component Id

ORG-NAV-001

Component Name

Navigation Bar

Component Type

organism

Description

Responsive top navigation bar handling branding, primary navigation links, and user actions. Supports states for public, authenticated, and admin contexts with adaptive mobile behaviors.

Design System Alignment

Implements layout tokens, typography scale, and semantic color usage for active/inactive states.

Design Tokens Used

--color-white
--color-gray-200
--color-primary-600
--spacing-4
--shadow-sm
--z-index-sticky

Responsive Specifications

Mobile Adaptations

  • Navigation links collapse into a full-width drawer/dropdown
  • Hamburger menu trigger appears on right
  • Header height reduces to 56px
  • User actions move inside the mobile menu or simplify

Tablet Adaptations

  • Spacing between links reduces
  • Secondary actions may group into a 'More' menu if crowded

Desktop Enhancements

  • Full horizontal link layout
  • Expanded user profile actions
  • Hover states for dropdowns

Accessibility Features

Keyboard Support

Logical tab order through links and actions. Skip-link target.

Screen Reader Support

Semantic

landmarks, aria-labels for icon buttons, hidden text for visual-only elements.

Focus Management

Visible focus rings on all interactive elements.

Aria Attributes

  • aria-label='Main Navigation'
  • aria-expanded (requires JS for dynamic state)
  • aria-current='page'

Color Contrast

Navigation text 4.5:1 against background

CSS Implementation

See embedded <style> block in the HTML section.

Implementation Notes

Embedded Css Architecture

BEM naming convention (e.g., .navbar__brand, .navbar__menu). CSS variables for easy theming.

Component Organization

Self-contained styling. Uses the 'Checkbox Hack' for pure CSS mobile menu interaction in this mockup (production should use JS).

Responsive Design

Media queries at 768px break point to toggle between desktop flex row and mobile column layout.

Accessibility Compliance

Semantic HTML5 tags (header, nav, ul, li, a, button).

Browser Support

Works in all modern browsers (Chrome, Firefox, Safari, Edge).

Usage Guidelines

When To Use

At the top of every page to provide global navigation and context.

Content Guidelines

Limit top-level links to 5-7 items. Use clear, short labels.

Placement Rules

Always fixed or sticky at the top of the viewport.

Customization Options

  • Variant classes: .navbar--admin (dark theme), .navbar--transparent (overlay)
  • Sticky positioning via .navbar--sticky

Testing Recommendations

Browser Testing

Verify hamburger toggle on mobile Safari and Chrome.

Device Testing

Test touch targets on small phones.

Accessibility Testing

  • Verify tab order includes mobile menu items when open
  • Check contrast of active link state

Performance Testing

Ensure layout shift (CLS) is zero during load.
